## Контекст для Claude Code

Мы делаем **ЭкоПульс** — простую внутреннюю админку для оператора-эколога, который собирает обращения о состоянии контейнерных площадок ТКО из двух источников (**мессенджер Макс** и **Яндекс Форма**) и доводит каждое до статуса. Бренд — зелёное сердце-смайлик 💚 (сайт `ecopulse.reo.ru`). Десктоп only, язык интерфейса русский, деловой тёплый тон.

Есть готовый рабочий **прототип на React** в папке `react-app/` (классический React + Babel из CDN, без сборщика) — он отражает целевые поведение и вёрстку 1-в-1. Нужно довести его до продакшена.

### Стек прототипа (что переносим на прод)
- React 18 + Babel-standalone в браузере, отдельные `.jsx` без сборки → перевести на **Vite + React + TypeScript** с прекомпиляцией.
- Стили — инлайновые объекты + `styles.css`. Дизайн-система «Аркадий CRM»: белый фон, синий акцент `#2A8AF0`, зелёный бренд `#1F8A5B`, Inter 13–14px, JetBrains Mono для чисел/ID/телефонов.

### Структура react-app/
```
index.html          — точка входа
styles.css          — шрифты, ховеры, keyframes (пульс сердца), скроллбары
data.js             — STATUS, SOURCE, INCIDENTS (моки, 13 шт.), хелперы, выгрузка в CSV
imgs.js             — фото-заглушки (data-URI)
App.jsx             — состояние, фильтрация/сортировка, переключение разделов
components/
  Sidebar.jsx        — бренд 💚, навигация (Инциденты / Настройки)
  Funnel.jsx         — воронка статусов (быстрый фильтр)
  FilterBar.jsx      — фильтры: источник + период (календарь «с—по»)
  IncidentsTable.jsx — таблица; замороженный блок «Дата·время·ID·телефон» слева, сортировка по заголовкам
  DetailDrawer.jsx   — карточка инцидента (выезжает справа, экран загрузки, смена статуса)
  Lightbox.jsx       — просмотр фото
  Settings.jsx       — профиль, сброс пароля, пользователи + приглашение по email
```

### Модель данных (инцидент)
`id, source('max'|'form'), status('new'|'found'|'none'|'exported'), fio, phone, region, city, street, coords, photoTime, photos(1-3), photoUrls[], msg(id сообщения Макса), dateRaw`.
Адрес из Макса приходит одной строкой — на бэкенде разложить на region/city/street (геокодер + ручная правка).

### Статусы и переходы
- `Новый` → «Инцидент обнаружен», «Нет инцидента»
- `Инцидент обнаружен` → «Новый», «Выгружен»
- `Нет инцидента` → «Новый», «Инцидент обнаружен»
- `Выгружен` → «Инцидент обнаружен», «Новый»

### Ключевые экраны
1. **Инциденты** — плотная таблица. Слева **замороженный** блок (sticky, тень): фото-миниатюра + дата/время + ID (моно-чип) + телефон; сортируется единым ключом дата+время. Дальше столбцы Регион · Город · Адрес · Координаты · Статус · Источник · Чат (у Макса — ссылка на сообщение). Сортировка по клику на заголовок (▲▼). Сверху — воронка статусов и фильтры (источник, период-календарь). Чекбоксы + массовая выгрузка в Excel и пометка «Выгружен».
2. **Карточка инцидента** — выезжает справа над таблицей (воронку/фильтры не закрывает), при открытии короткий экран загрузки с пульсом 💚. Сверху — контекстные кнопки смены статуса; фото (одиночное не растягивается, клик → лайтбокс); все поля; для Макса — «Открыть сообщение в Максе».
3. **Настройки** — профиль (ФИО + email), сброс пароля (одно поле + «Сбросить»), пользователи и доступ, добавление пользователя с отправкой приглашения на email.

### Задача для прода (mock → real)
- БД + API для обращений, пагинация/виртуализация (таблица должна тянуть 1000+).
- Интеграция с **Макс** (бот/webhook): приём фото + текста, парсинг адреса и «Радар №», ссылка на сообщение.
- Интеграция с **Яндекс Формами** (webhook): создание обращения, приём фото.
- Хранилище фото (S3/диск), превью + полноразмер вместо заглушек.
- Настоящая выгрузка `.xlsx` (сейчас CSV `;` + BOM).
- Аутентификация (email/пароль, сессии, сброс), приглашения по email с токеном, роли admin/user на сервере.
- Геокодер адресов; опц. дедупликация обращений по площадке; аудит смены статусов.
- Решить судьбу поля «Наличие баков раздельного сбора» (есть в данных формы, в таблице скрыто).
