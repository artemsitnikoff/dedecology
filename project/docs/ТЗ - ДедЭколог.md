# ТЗ — ДедЭколог

**Админка для сбора и обработки обращений о состоянии площадок ТКО**

Версия документа: 1.0 · Прототип: React SPA (`react-app/`) + standalone-сборка (`ДедЭколог (standalone).html`)

---

## 1. О продукте

**ДедЭколог** — простая внутренняя админка для оператора (эколога/инспектора), который собирает обращения о состоянии контейнерных площадок ТКО из двух источников и доводит каждое до результата (инцидент подтверждён / не подтверждён / выгружен в отчёт).

**Пользователь:** оператор за ноутбуком в смену. Один экран — список обращений, плотная таблица, быстрые фильтры и выгрузка в Excel.

**Источники обращений:**
1. **Макс** (мессенджер) — пользователь присылает фото площадки и адрес текстом. Пример сообщения:
   `Самарская область, городской округ Кинель, посёлок городского типа Усть-Кинельский, Бульварная улица, 18 (Радар №116320)` + фото.
   У такого обращения должна быть **ссылка на исходное сообщение** в Максе.
2. **Яндекс Форма** — структурированная форма со следующими полями:
   - Полный адрес — текст (пример: `Великий Новгород, ул. Радужная 15`)
   - Координаты — текст с числами (пример: `55.859624, 37.663597`)
   - Время фотофиксации — текст
   - Наличие баков для раздельного сбора — да/нет *(в текущей версии админки в таблице не отображается, см. §11)*
   - 1–3 горизонтальных фото площадки ТКО — изображения
   - ФИО — текст

## 2. Глоссарий

| Термин | Значение |
|---|---|
| Обращение / инцидент | Одна запись в системе (из Макса или Формы) |
| Источник | `Макс` или `Яндекс форма` |
| Статус | Стадия обработки обращения (см. §6) |
| Воронка | Панель статусов сверху таблицы (быстрый фильтр по статусу) |
| Фотофиксация | Дата/время съёмки площадки, переданные в обращении |

---

## 3. Модель данных

### 3.1. Инцидент (обращение)

| Поле | Тип | Источник | Примечание |
|---|---|---|---|
| `id` | string | система | Уникальный идентификатор |
| `source` | enum: `max` \| `form` | система | Канал поступления |
| `status` | enum (см. §6) | оператор | По умолчанию `Новый` |
| `fio` | string | оба | ФИО заявителя |
| `region` | string | разбор адреса | Регион/субъект (напр. «Самарская область») |
| `city` | string | разбор адреса | Город/населённый пункт (напр. «г. Кинель») |
| `street` | string | разбор адреса | Улица, дом, доп. (напр. «Бульварная улица, 18 (Радар №116320)») |
| `coords` | string | оба | Координаты «широта, долгота» |
| `photoTime` | string | оба | Дата и время фотофиксации, формат `ДД.ММ.ГГГГ, ЧЧ:ММ` |
| `photos` | number | оба | Кол-во фото (1–3) |
| `photoUrls` | string[] | оба | Ссылки/данные фотографий *(в прототипе — заглушки)* |
| `msg` | string | Макс | ID исходного сообщения → ссылка `https://max.ru/m/{msg}` |
| `dateRaw` | string | система | Дата/время поступления `ГГГГ-ММ-ДД ЧЧ:ММ` |
| `bins` | boolean | Форма | Наличие баков раздельного сбора *(см. §11)* |

> **Разбор адреса.** Из Макса адрес приходит одной строкой — на бэкенде её нужно разложить на `region` / `city` / `street`. В прототипе поля заданы явно. Для продакшена — парсер адреса (напр. через геокодер Яндекса/Dadata) с возможностью ручной правки оператором.

### 3.2. Пользователь системы (раздел «Настройки»)

| Поле | Тип | Примечание |
|---|---|---|
| `id` | string | |
| `fio` | string | |
| `email` | string | На него уходит приглашение |
| `role` | enum: `admin` \| `user` | Админ — полный доступ, пользователь — ограниченный |
| `status` | enum: `active` \| `invited` | `invited` — приглашение отправлено, вход не выполнен |

## 4. Экран «Инциденты» (главный)

Структура сверху вниз:

1. **Шапка:** заголовок «Инциденты», счётчик (`N обращений` или `Показано X из N` при активных фильтрах), кнопка **«Выгрузить всё»** (экспорт текущей выборки).
2. **Поиск:** строка поиска по ФИО, региону, городу, улице, координатам, id сообщения. Справа подсказка про сортировку.
3. **Воронка статусов** (§6) — быстрый фильтр по статусу.
4. **Панель фильтров** (§5).
5. **Панель массовых действий** — появляется при выделении строк (§7).
6. **Таблица** (§4.1) либо пустое состояние.

### 4.1. Таблица — столбцы (строго в этом порядке)

| # | Столбец | Содержимое | Сортировка |
|---|---|---|---|
| — | Чекбокс | Выделение строки / «выделить все» в шапке | — |
| 1 | **Фото** | Миниатюра первого фото + бейдж с числом фото; клик → лайтбокс | нет |
| 2 | **Дата** | Дата фотофиксации | да |
| 3 | **Время** | Время фотофиксации | да |
| 4 | **Регион** | Область/субъект | да |
| 5 | **Город** | Населённый пункт | да |
| 6 | **Адрес** | Улица/дом (растягивается) | да |
| 7 | **Координаты** | Моноширинно | нет |
| 8 | **Статус** | Цветной бейдж с точкой | да |
| 9 | **Источник** | Бейдж `Макс` / `Яндекс форма` | да |
| 10 | **Чат** | Для Макса — ссылка «Макс» на сообщение; для Формы — пусто | нет |

**Сортировка — по клику на заголовок столбца** (как в Глафире): первый клик — по убыванию, повторный — по возрастанию; индикатор ▲/▼ в заголовке, активное направление подсвечено акцентом. Сортировка по умолчанию — по дате/времени фотофиксации, убывание.

Клик по строке (кроме чекбокса/ссылки/фото) открывает **карточку инцидента** (§8).

---

## 5. Фильтры

Панель под воронкой, всегда раскрыта:

- **Источник:** мультивыбор чипами `Макс` / `Яндекс форма`.
- **Период:** два поля даты (нативный календарь) **«с — по»**, фильтрует по дате фотофиксации; крестик сбрасывает диапазон.
- **Сбросить** — появляется при активных фильтрах, сбрасывает источник + статус + период.

Поиск (§4) комбинируется с фильтрами по «И».

---

## 6. Статусы и воронка

Четыре статуса (lifecycle обращения):

| Статус | Цвет | Смысл |
|---|---|---|
| **Новый** | синий | Только поступило, не обработано |
| **Инцидент обнаружен** | красный | Нарушение подтверждено |
| **Нет инцидента** | серый | Проверено, нарушения нет |
| **Выгружен** | зелёный | Включено в отчёт/выгрузку |

**Воронка** сверху таблицы — 1-в-1 как в Глафире: чип `Все`, далее пайплайн `Новый → Инцидент обнаружен`, затем через разделитель терминальные `Выгружен` (зелёный) и `Нет инцидента` (красный). У каждого — счётчик. Клик по чипу фильтрует таблицу по статусу (одиночный выбор), `Все` — сброс.

Статус меняется вручную в карточке инцидента (§8). Массовая простановка `Выгружен` — в панели массовых действий (§7).

---

## 7. Выделение и выгрузка в Excel

- Чекбокс в каждой строке + «выделить все» в шапке таблицы.
- При наличии выделения появляется **панель массовых действий**: «Выбрано: N», кнопки:
  - **Выгрузить в Excel** — экспорт выбранных.
  - **Пометить «Выгружен»** — массовая смена статуса выбранных на `Выгружен`.
  - **Снять выделение**.
- Кнопка **«Выгрузить всё»** в шапке экспортирует текущую (отфильтрованную) выборку.

**Формат выгрузки.** В прототипе — CSV (разделитель `;`, UTF-8 с BOM, открывается в Excel с кириллицей). Колонки: ФИО, Источник, Статус, Регион, Город, Адрес (улица), Полный адрес, Координаты, Дата фотофиксации, Время фотофиксации, Кол-во фото, Ссылка на сообщение, Поступило.

> **Продакшен:** заменить на настоящий `.xlsx` (напр. SheetJS/ExcelJS) с форматированием, при необходимости — со встроенными фото или ссылками на них.

## 8. Карточка инцидента

Выезжает справа (drawer) по клику на строку. Содержит:

- Бейджи источника и статуса, полный адрес заголовком.
- Фотографии площадки (1–3) — клик открывает лайтбокс.
- Поля: ФИО, Регион, Город/н.п., Адрес, Координаты, Дата/Время фотофиксации, Кол-во фото, Источник, Поступило.
- Для Макса — кнопка **«Открыть сообщение в Максе»** (ссылка).
- **Смена статуса** — чипы всех 4 статусов, активный выделен.

### 8.1. Лайтбокс (просмотр фото)

Полноэкранный просмотр фото с затемнением. Стрелки ◀ ▶ листают (если фото несколько), счётчик `1 / N`, подпись с адресом, клик по фону/крестик — закрыть. Открывается из таблицы (по миниатюре) и из карточки.

---

## 9. Раздел «Настройки»

Пункт в сайдбаре под «Инцидентами». Два блока:

### 9.1. Профиль
- **ФИО** — редактируемое поле.
- **Email для входа** — только для чтения.
- Кнопка **«Сохранить»**.
- **Смена пароля:** одно поле «Новый пароль» + кнопка **«Сбросить»** (устанавливает новый пароль). Подтверждение и ввод текущего пароля не требуются.

### 9.2. Пользователи и доступ
- Список пользователей: аватар-инициалы, ФИО, email, **роль** (Администратор/Пользователь), **статус** (Активен/Приглашён).
- **Добавление пользователя:** ФИО + email + выбор доступа → **«Отправить приглашение»**. Пользователь добавляется со статусом `Приглашён`, на email уходит приглашение со ссылкой для входа (как у Глафиры). Показывается подтверждение отправки.
- Не-администраторов можно удалить (иконка корзины); у администратора — иконка замка (удаление недоступно).

## 10. Архитектура прототипа (для разработки)

React SPA по образцу проекта «Глафира»: классический React + Babel, отдельные файлы компонентов, `ReactDOM.createRoot`. Стили — инлайновые объекты + общий `styles.css` (ховеры, анимации, шрифты). Без сборщика — для прода рекомендуется перевести на Vite + предкомпиляцию JSX.

```
react-app/
  index.html                  — точка входа, подключает React/Babel, стили, данные, компоненты
  styles.css                  — шрифты (Inter, JetBrains Mono), ховеры, keyframes, скроллбары
  imgs.js                     — фото площадок в data-URI (заглушки)
  data.js                     — STATUS, SOURCE, INCIDENTS (моки), хелперы, выгрузка в CSV
  App.jsx                     — корневой компонент: состояние, фильтрация/сортировка, роутинг вида
  components/
    Sidebar.jsx               — бренд, навигация (Инциденты / Настройки), карточка пользователя
    Funnel.jsx                — воронка статусов
    FilterBar.jsx             — фильтры (источник, период-календарь)
    IncidentsTable.jsx        — таблица + строки + сортировка по заголовкам
    DetailDrawer.jsx          — карточка инцидента
    Lightbox.jsx              — просмотр фото
    Settings.jsx              — раздел «Настройки»
```

Цветовая палитра и типографика — дизайн-система «Аркадий CRM» (белый фон, синий акцент `#2A8AF0`, плотный десктоп, Inter 13–14px, JetBrains Mono для чисел). Семантика статусов: серый → синий → жёлтый → зелёный.

> Также есть `ДедЭколог (standalone).html` — та же админка в одном самодостаточном файле (всё внутри, работает офлайн) для демонстрации без сервера.

## 12. Нефункциональные требования

- Десктоп only (мин. ширина ~1180px), браузеры Chrome/Edge/Firefox актуальных версий.
- Язык интерфейса — русский, тон деловой и тёплый, sentence case в заголовках/кнопках.
- 24-часовой формат времени; даты `ДД.ММ.ГГГГ`; координаты моноширинно.
- Без эмодзи в основном UI (кроме знака бренда 👴).
- Производительность: таблица должна оставаться отзывчивой на 1000+ обращений (виртуализация при необходимости).
