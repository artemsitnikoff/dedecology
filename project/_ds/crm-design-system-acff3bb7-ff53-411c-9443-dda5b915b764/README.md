# Аркадий CRM — Design System

## О продукте

**Аркадий CRM** — CRM-система для ногтевых салонов с AI-администратором.
Аркадий — это виртуальный администратор, который ведёт календарь, принимает и переносит брони, ведёт досье клиентов и сам ставит себе задачи на следующий контакт.

Целевая аудитория: владельцы и администраторы маленьких/средних салонов красоты (1–10 мастеров), которые работают на ноутбуке/десктопе в смену и переписываются с клиентами в Telegram/WhatsApp/Instagram.

### Продуктовая позиция

| | |
|---|---|
| Жанр | Десктопная CRM с мессенджером |
| Вайб | Telegram Desktop × Kommo × Bitrix24 — лёгкое, плотное, на белом фоне |
| Ключевые экраны MVP | Дашборд, Лиды (канбан), Чаты, Календарь |
| Tech stack (для UI Kit) | React + Tailwind-like utility CSS |

### Экраны MVP

1. **Дашборд** — статистика (total leads, by status/source, conversion rate), графики Chart.js, ближайшие задачи AI-администратора
2. **Лиды** — канбан 4 колонки (`Новый → Контакт → Квалифицирован → Выигран`), drag-n-drop через `vuedraggable`
3. **Чаты** — мессенджер в стиле Telegram Desktop (список чатов слева + переписка справа), real-time через WebSocket
4. **Календарь** — `FullCalendar` (месяц / неделя / день), drag-n-drop событий, модалка создания/редактирования

## Источники

На момент создания дизайн-системы пользователь **не приложил** ни кодовой базы, ни Figma, ни скриншотов. Сборка сделана на основе текстового брифа и жанровых референсов:

- **Telegram Desktop** — лёгкий мессенджер, белый фон, синий акцент, плотные списки
- **Kommo / amoCRM** — канбан лидов, цветные статусы, чипы, dashboard-карточки
- **Bitrix24** — плотная CRM с боковым навбаром и насыщенным контентом

> ⚠️ Дизайн-система **оригинальная**, а не копия перечисленных продуктов. Она использует общие жанровые конвенции (десктопный CRM-мессенджер, белый фон, синий акцент), но цвета, типографика, иконография и компоненты — собственные.

Когда появятся реальные ассеты (Figma, кодовая база, гайдлайн бренда) — система обновится.

## CONTENT FUNDAMENTALS

### Язык

Основной язык интерфейса — **русский**. Английские термины (CRM, lead, dashboard) допустимы внутри текстов, но интерфейсные элементы — на русском.

### Тон

**Деловой, тёплый, без официоза.** Аркадий — это коллега, который помогает, а не корпоративный продукт, который что-то от вас требует.

| Не пишем | Пишем |
|---|---|
| «Уважаемый пользователь, выполните вход в систему» | «Войти в Аркадий» |
| «Произошла ошибка. Обратитесь к администратору» | «Не получилось сохранить. Попробуйте ещё раз» |
| «Создание новой записи о клиенте» | «Новый клиент» |
| «Запись успешно сохранена» | «Сохранено» |

### Обращение

- К пользователю — на **«вы»** (со строчной)
- От имени AI — Аркадий говорит **«я»**: «Я записал Аню на 14:00», «Поставлю себе напоминание перезвонить»
- Никакого «Мы рады приветствовать вас в системе»

### Регистр

- **Sentence case** для заголовков и кнопок: «Новый лид», «Сохранить изменения», «Календарь»
- НЕ Title Case, НЕ ВСЁ КАПСОМ
- Исключения: статусы канбана пишутся с большой буквы как имена собственные («Новый», «Контакт», «Квалифицирован», «Выигран»)

### Эмодзи

**Не используем** в основном UI. Допустимы только:
- В сообщениях клиентов (входящий контент)
- В уведомлениях самого Аркадия как акцент: «Готово ✓» — но и тут предпочтительнее иконка

### Числа, даты, время

- Время — 24-часовой формат: `14:00`, не `2 PM`
- Даты в плотных таблицах — `25.04`, в карточках — `25 апреля`, в полных — `25 апреля 2026`
- Деньги — `4 500 ₽` (тонкий пробел между разрядами, символ ₽ через пробел)
- Телефоны — `+7 999 123-45-67`

### Пустые состояния

Короткие, по делу, с действием:

> **Пока никого**
> Лиды появятся, как только клиент напишет в чат
> [Подключить канал]

Не «Список пуст», не «Здесь будут отображаться ваши лиды».

### Микрокопия Аркадия

**Аркадий — это сотрудник вашего салона, не «ассистент» и не «бот».** Он работает наравне с админом и мастерами: ведёт календарь, пишет клиентам, ставит задачи. Поэтому:

- Говорит от первого лица как коллега: «**я записал** Аню», «**я перенёс**», «**я напомню**»
- Не спрашивает разрешения на каждое действие — отчитывается о сделанном
- В UI его статус — «онлайн» как у любого сотрудника
- В отчётах его действия включены в общую статистику работы команды

Примеры:
- «Я записал Аню Петрову на маникюр, 25 апреля 14:00»
- «Перенёс Машу с пятницы на понедельник — у неё не получалось»
- «Я напомню себе перезвонить через 3 дня»
- «Это новый клиент. Завёл карточку, передаю вам»
- «Сегодня я закрыл 4 записи и поставил 7 задач»

## VISUAL FOUNDATIONS

### Общая философия

**Лёгкое, плотное, на белом фоне.** Без градиентов, без больших теней, без декоративных иллюстраций. Интерфейс должен ощущаться как рабочий инструмент, в котором проводят 6+ часов в день: спокойный, не отвлекающий, с чёткой иерархией через типографику и тонкие границы.

### Цвет

Основной фон — **белый** (`#FFFFFF`). Поверхности — почти белые с лёгким холодным оттенком. Акцент — спокойный синий (`#2A8AF0`), не «кислотный», без свечения.

| Слой | Цвет |
|---|---|
| Фон приложения | `#FFFFFF` |
| Сайдбар / панели | `#F4F6F8` |
| Бордюры | `#E6E9EC` |
| Текст основной | `#0F1620` |
| Текст вторичный | `#5B6573` |
| Акцент | `#2A8AF0` |

Семантика статусов лида: серый → синий → жёлтый → зелёный (нейтральный → активный → внимание → успех).

См. `colors_and_type.css` — там лежат все CSS-переменные.

### Типографика

- **Inter** (variable) — основной шрифт UI. Используем 13–14px как базу (десктопная плотность).
- **JetBrains Mono** — числа в таблицах и моноширинные элементы (айди, телефоны в плотных списках).

> ⚠️ Брендового шрифта пользователь не предоставил. Использован `Inter` как ближайший подходящий для жанра. **Если есть фирменный шрифт — пришлите файлы, заменю.**

Размеры:
- `12px` — caption, лейблы, метаданные
- `13px` — таблицы, плотные списки
- `14px` — основной текст интерфейса (кнопки, поля)
- `16px` — заголовки секций
- `20–24px` — заголовки экранов / большие числа на дашборде

### Спейсинг

Шкала кратна **4px**: `4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48`. Для плотных десктопных интерфейсов основной gap — `8px` или `12px`.

### Скругления

- `4px` — мелкие чипы, статусы
- `6px` — кнопки, поля ввода (стандарт)
- `8px` — карточки лидов, попапы
- `12px` — модалки, крупные карточки дашборда
- `50%` — аватары

### Тени

**Минимально.** Только для всплывающих слоёв.

| Уровень | Использование | Тень |
|---|---|---|
| `shadow-0` | плоские поверхности (90% UI) | — |
| `shadow-1` | карточки в hover | `0 1px 2px rgba(15, 22, 32, .06)` |
| `shadow-2` | дропдауны, тултипы | `0 4px 12px rgba(15, 22, 32, .08)` |
| `shadow-3` | модалки | `0 12px 32px rgba(15, 22, 32, .12)` |

Iner shadow не используем. Цветных теней нет.

### Бордюры

Чаще, чем тени. `1px solid #E6E9EC` — основной разделитель. Внутри карточек — нет; между элементами в списке — да.

### Фоны и поверхности

- Без иллюстраций, без узоров, без градиентов
- Допустим тонкий чекерборд (`#F8F9FB` 1px каждые 8px) на пустых дроп-зонах канбана
- Аватары без фото — заливка одним из 8 brand-friendly цветов + инициалы

### Анимация

- Микро-переходы: `120ms ease-out` для hover, `200ms cubic-bezier(.2, .8, .2, 1)` для входящих элементов
- Канбан drag — лёгкая `scale(1.02)` + `shadow-2` под курсором
- Никаких bounce, никаких больших spring-эффектов
- Сообщения в чате появляются с `fade + translateY(4px)`, 180ms

### Состояния hover / press

- **Hover** на кнопке/строке списка — фон темнеет на 4% (на белом → `#F4F6F8`, на сером → `#ECEFF2`)
- **Hover** на акцентной кнопке — `#2A8AF0` → `#1F7AE0`
- **Press** — фон ещё на 4% темнее, без `scale`-эффекта (это десктоп)
- **Focus** — кольцо `2px` цвета `rgba(42, 138, 240, .35)` снаружи элемента

### Прозрачность и блюр

- Блюр **не используем** (это не iOS)
- Полупрозрачность — только для disabled (`opacity: .5`) и для оверлея модалки (`rgba(15, 22, 32, .4)`)

### Карточки

Стандартная карточка:
```
background: #FFFFFF
border: 1px solid #E6E9EC
border-radius: 8px
padding: 12px
shadow: none (hover → shadow-1)
```

Карточка лида в канбане — компактнее (10px padding, 6px radius).

### Layout

- Боковой навбар фиксирован слева, ширина `64px` (иконки) или `240px` (с подписями) — переключается
- Шапка экрана `52px`, фиксирована сверху
- Контент скроллится внутри центральной области; шапка и сайдбар — нет
- Минимальная ширина приложения — `1180px` (десктоп only для MVP)

## ICONOGRAPHY

### Подход

**Линейные иконки, stroke 1.5px, 20×20.** Похожи на Lucide/Feather, но без декоративных деталей. Цвет — `currentColor`, чтобы наследовался от текста.

### Источник

Используется **Lucide Icons** (CDN: `lucide.dev`) как ближайший по стилю опенсорсный набор. Подключаем через `<i data-lucide="...">` или импортом SVG-файлов.

> ⚠️ Это **подстановка**: своего иконсета у бренда нет. Когда появится — заменим. Lucide хорошо подходит, потому что: (а) современный stroke-style, (б) большой охват (ноготь, календарь, чат, бриф — всё есть), (в) лицензия ISC.

Локальная копия ключевых иконок в `assets/icons/` для оффлайн-использования.

### Размеры

- `16px` — внутри плотных списков, чипов
- `20px` — стандарт (кнопки, навигация)
- `24px` — крупные акценты в шапке

### Эмодзи и юникод

- Эмодзи в UI — **нет** (см. CONTENT FUNDAMENTALS)
- Юникод-символы — допустимы только функциональные: `✓` (галочка в чате-«прочитано», как в Telegram), `→`, `·` (разделитель в метаданных)

### Логотип

Простой словесный знак: **«Аркадий»** — Inter Bold, с маленьким акцентным маркером (точка/каплевидная форма) перед словом. Файл: `assets/logo.svg`. Монохромный, на белом — `#0F1620`, на тёмном — `#FFFFFF`.

## Каверы и ограничения

- **Шрифт Inter** — подстановка. Бренд не предоставил фирменный шрифт; Inter подходит по жанру (Telegram Desktop, Kommo). При появлении бренд-шрифта — заменить переменные в `colors_and_type.css`.
- **Lucide Icons** — подстановка. Локально в `assets/icons/` нет — иконки нарисованы SVG-шками внутри `primitives.jsx` в Lucide-style. При интеграции — перейти на CDN `lucide.dev`.
- **Логотип** — сделан с нуля как простая блочная марка с каплевидной формой (отсылка к ногтю/капле лака). Не финальный — нужен бренд-знак от пользователя.
- **Канбан и Календарь** в UI kit реализованы через native HTML5 DnD, а не `vuedraggable` / `FullCalendar`. Это визуальный референс — production-стек подключается отдельно.
- **WebSocket в чате** не реализован, но UI рассчитан на потоковые сообщения (анимация fadeIn у `.msg`, индикатор «в сети» в шапке).
